Relative, absolute, and fixed positioning

Understanding CSS positioning

Magda Ang

May 24th, 2014

Positioning seems like an easy concept to understand; however, is a little more complicated than it first appears. First thing to understand: every single element on a web page is a block. Once we get that, we can talk about how we can use positioning to get the blocks of pixels exactly where we want them to go and have fun, creative designs.

The CSS position property takes 4 values:

  1. Static
  2. Relative
  3. Absolute
  4. Fixed

Static

Is the default position for any element on a web page. If we apply positioning rules like top or left to an element that has a static position, nothing will happen, those rules are ignored.

Relative

Is similar to static except you're saying "I'd like that where it normally would be except a little to the left". It's a way to take an element without removing it from the natural flow of the page and just push o pull it to any of the four positioning properties, which are: top, right, bottom, and left. And talking about this, here comes my "aha moment": to use these properties, you need to think of them as offset properties, meaning that an element positioned right: 20px, is not moved right 20px. It's right side is offset from the right side of the window 20px, in other words, it moves to the left!! And this applies to the other three too.

Absolute

This tells the browser that whatever element to be positioned should be removed from the normal flow of the document and will be placed in an exact location on the page. You're basically saying "I want this element here and I don't care what is around it". It is also taken out of the normal flow, it won't affect how the elements before or after it in the HTML are positioned on the web page.

Absolutely positioned elements are positioned relative to the first parent element that has a position other than static applied to it. If there is no parent element up that meets this condition, the absolutely positioned element is positioned relative to the document window.

Fixed

Fixed elements are always positioned relative to the browser window. No matter how far down the page you scroll, you will always see it.
